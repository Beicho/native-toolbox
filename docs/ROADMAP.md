> 从 33 个工具到 **200 个**，每一个都做完整 —— 不毛坯、不凑数、默认离线。
> 本文档是唯一路线事实源：功能定义、云端架构、版本分期、验收标准都在这里。

## 2. 设计原则（质量红线）

1. **一个工具 = 一个完整场景**。同族能力合并成一个工具（如「文本处理」含去重/排序/大小写/批量替换），绝不拆开凑数。本表已主动合并的例子：文本哈希与文件哈希 → 「哈希与校验」；翻译与词典 → 一个工具；尺子与量角器 → 「屏幕测量」。
2. **不毛坯**。每个工具的达标线见 §9：空态、输入校验、错误提示、结果可复制/分享/保存、深浅色、CI 遍历零崩溃，一项不缺才算完成。
3. **默认离线是卖点**。需要联网的工具在图标角标和页面首行明示「联网 · 服务由 Astro 云提供」，断网时展示最近缓存而不是白屏。
4. **文案讲人话**。所有用户可见文字只说「能干什么、好在哪」，技术名词一律不出现（原则见 §8）。

## 3. 首页信息架构（200 个工具怎么放不乱）

当前 7 分类平铺宫格到 200 个会翻车，改造为三层结构：

- **首层**：搜索框置顶（拼音首字母模糊匹配）+「常用」自动区（使用计数前 8）+「猜你需要」情境卡（充电时→电池信息；连新 WiFi→WiFi 分析，纯本地规则）。
- **分类导航**：顶部横向玻璃 chips（13 分类），点击锚点滚动；宫格改分组粘性表头 + 右缘字母/分类快滚索引条（iOS 通讯录式）。
- **全局**：每个工具支持收藏（长按）、最近使用时间线；分享接入与桌面快捷方式沿用 v0.3 的 ShareBus/深链体系，全部 200 个路由自动注册。
- ToolRegistry 仍是唯一事实源，新增 `tags`（搜索别名）、`requiresNetwork`、`permissions` 字段，首页/搜索/角标/权限说明全部由注册表驱动。

## 4. 云端架构：`astro-api` Worker

**一个 Worker，多路由，绑自有域名并开 CDN。** 客户端只认 `https://api.606333.xyz`。

| 路由 | 供给 | 数据源 / 存储 | 缓存策略 |
|---|---|---|---|
| `/v1/update` | 检查更新反代 | GitHub Releases API | CDN 10 min |
| `/v1/exchange` | 汇率 | Cron Trigger 定时抓取 → KV | CDN 1h + stale-while-revalidate |
| `/v1/weather` `/v1/air` | 天气/空气 | Open-Meteo 反代 | CDN 30 min（按坐标网格取整） |
| `/v1/ip` | IP 归属/ASN | CF request.cf + 离线库 | CDN 1d（按 IP） |
| `/v1/whois` `/v1/ssl` `/v1/sitecheck` | Whois/证书/网站体检 | Worker 现场探测 | CDN 1h |
| `/v1/dns` | DoH 兜底 | Cloudflare DoH 反代 | 不缓存 |
| `/v1/today` `/v1/holiday` `/v1/hitokoto` | 历史上的今天/假期/一言 | 预置数据 → KV | CDN 1d |
| `/v1/translate` | 翻译与词典 | Workers AI（m2m100） | KV 记忆热句 |
| `/s/:code` | 短链跳转 + 生成 | KV（含访问计数） | 短链页不缓存 |
| `/mail/*` | 临时邮箱 | Email Routing 收 `*@606333.xyz` → Worker → KV（TTL 1h） | 不缓存 |

- **域名**：`api.606333.xyz`、`s.606333.xyz`（短链）、`606333.xyz` MX 指 CF Email Routing。全部 orange cloud（CDN 开启）+ Cache Rules。
- **部署**：优先 wrangler；本机 PRoot 跑不动就直接走 CF REST API 上传脚本（Global Key 已有全权限，`X-Auth-Email + X-Auth-Key`）。Worker 源码放本仓库 `worker/` 目录，与 App 同版本管理。
- **约定**：统一 JSON envelope `{ok, data, err}`；轻量 UA 校验 + 每 IP 限频防白嫖；客户端超时 3s 自动降级到本地缓存。
- **App 侧**：引入 OkHttp（仅此一个网络依赖），统一 `AstroApi` 单例；「无统计 SDK、不收集数据」承诺不变。

## 5. 权限与隐私策略

- 安装即用零权限；相机（扫码/证件照/文档扫描）、麦克风（录音/分贝/调音）、位置（WiFi 分析/停车/天气）、通知（提醒类）、NFC/蓝牙等全部**在工具页首次使用时**申请，页面内一句话解释用途，拒绝后功能优雅降级。
- 敏感数据（密码保险箱、经期记录、私密相册、记账）一律本地 AES-GCM 加密 + 可选生物识别锁，永不上传；导出备份也是加密容器。
- 关于页新增「隐私说明」：一屏说清哪些工具联网、传什么、不传什么。

## 8. 文案整改（v0.4 随版落地）

- **App 关于页**:只留四件套 —— 版本与检查更新、功能反馈入口、隐私说明、开源致谢；删除一切框架/库名罗列。

## 9. 完成的定义（DoD，每个工具逐条过）

1. 空态有引导、非法输入有提示、长任务有进度、失败可重试;
2. 结果可复制/分享/保存至少其一;深浅色与动态字号正常;
3. 联网工具断网降级到缓存并标注时间;权限拒绝后有替代路径或清晰说明;
4. 进 ToolRegistry(搜索别名齐全)、分享接入路由、CI 遍历零崩溃、截图验收通过;
5. 副标题与页内文案过一遍 §8 原则。

## 10. 风险与取舍备忘

- mp3 编码系统不支持 → 音频转换如实只标 wav/m4a/flac/ogg，不虚标;
- 语音离线识别依赖系统离线包 → 首次引导下载,不可用则热切云端 **[网]**;
- 临时邮箱/短链有滥用风险 → 限频 + TTL 自毁 + 保留关停开关;
- 健康类(心率/视力)全部标注「仅供参考,非医疗设备」;
- 观察池里依赖不稳定数据源的一律不进主表，宁缺毋滥。
